01 How the curves are fit

Two families of curve are on offer, and they answer slightly different questions. A polynomial fit assumes a single global shape; a LOESS smoother makes no such assumption and simply lets the data bend the line locally.

Least-squares polynomiallinear / quad / cubic

Pick a degree d and choose the coefficients that minimise the total squared vertical distance between the points and the curve.

ŷ = β0 + β1x + β2x2 + … + βdxd

The coefficients drop out of the normal equations — one tidy linear system. A straight line is just degree 1; the U-shaped and S-shaped panels need degree 2 or 3.

LOESSlocal regression

For every point along the x-axis, fit a tiny weighted line using only the nearest fraction of the data — the span. Stitch those local fits together into one smooth curve.

weight(x) = (1 − |u|3)3,   u = dist / span

A small span hugs the data and wiggles; a large span is stiff and smooth. Drag the slider above to feel the trade-off.

Each panel reports R² — the share of the y-variable's variance the curve accounts for. It rises as you add degrees, but a higher number is not automatically a better model: an over-flexible curve chases noise.

02 The same thing in R

This whole figure is a styled version of two base-R idioms — pairs() for the matrix and lm() / loess() for the curves.

R
# scatterplot matrix with a smoother drawn on every panel
pairs(mtcars[, c("mpg", "hp", "wt", "disp")],
      lower.panel = panel.smooth)

# fit a curve to one pair -- quadratic least squares
fit <- lm(mpg ~ poly(hp, 2), data = mtcars)
summary(fit)$r.squared        # the R-squared shown in each panel

# a cubic is just one more degree
fit3 <- lm(mpg ~ poly(hp, 3), data = mtcars)

# ...or a local smoother instead of a global polynomial
lo <- loess(mpg ~ hp, data = mtcars, span = 0.7)
