K‑NEAREST NEIGHBORS

The simplest classifier in the playbook — no equations to memorize, no training phase, just a vote from whichever data points are standing closest.

01
The Playbook

What On Earth Is KNN?

KNN doesn't learn a model. It memorizes the training set and, at prediction time, asks the k closest points to vote.

Here's the whole idea: to classify a new flower, measure how far it is from every flower in your training data. Keep the k closest ones. Whichever species shows up most among those k neighbors wins — that's the prediction. That's it. That's the whole algorithm.

Most machine learning methods spend a long "training" phase crunching numbers to build a formula. KNN skips all of that — its training is literally just saving the data. All the real work happens later, when you ask it to classify something new. That's why people call KNN a lazy learner — it puts off the work until you actually need an answer.

DeNeighbor's First Rule

KNN decides "who's closest" using raw numerical distance. If one feature is measured in big numbers (say, height in millimeters: 1500–1900) and another in small ones (leaf count: 1–5), the big-numbered feature completely takes over. Always put your features on the same scale first so every one of them gets a fair say in the vote.

03
Training Camp

Load, Split, And Scale

R already comes with the iris dataset built in — just type iris and there it is. We pick out the two sepal measurements we want, randomly split the rows into a training group (which the model learns from) and a test group (which we'll use to check how well it did), and we're ready to go.

# The iris dataset is already built into R
data(iris)

# Pick just the two measurements we want
X <- iris[, c("Sepal.Length", "Sepal.Width")]
y <- iris$Species

# Randomly pick 75% of rows for training, the rest for testing
set.seed(0)
train_rows <- sample(1:nrow(iris), size = 0.75 * nrow(iris))

X_train <- X[ train_rows, ];  y_train <- y[ train_rows]
X_test  <- X[-train_rows, ];  y_test  <- y[-train_rows]
04
The Formation

Building The Classifier

Two steps. First, scale() puts each feature on the same footing (mean 0, spread of 1) so neither one bullies the distance calculation. Second, knn() from R's class package does the whole classification in one line. Starting formation: k = 11 neighbors.

# Step 1 — put both features on the same scale
X_train_scaled <- scale(X_train)

# Scale the test set using the SAME means and spreads from training
X_test_scaled  <- scale(X_test,
                        center = attr(X_train_scaled, "scaled:center"),
                        scale  = attr(X_train_scaled, "scaled:scale"))

# Step 2 — run KNN with 11 neighbors
library(class)
predictions <- knn(train = X_train_scaled,
                   test  = X_test_scaled,
                   cl    = y_train,
                   k     = 11)

# How often did we get it right?
mean(predictions == y_test)
05
The Film Room

Uniform vs Distance Weighting

Every neighbor gets a vote, but the weights parameter decides how loud that vote is. Uniform gives every one of the k neighbors an equal say. Distance gives each neighbor a vote proportional to 1 / distance — closer neighbors matter more.

The two pictures look similar overall, but look at the edges — the distance plot carves out tighter pockets around a few isolated versicolor points. That's because nearby neighbors get a louder vote, so even one or two close-by points can pull the prediction their way. Uniform weighting, where every neighbor gets the same vote, smooths those little pockets away.

DeNeighbor's Audible

Go uniform when the three groups are roughly evenly spread out and you want a clean, smooth boundary. Go distance when some groups are tightly clustered and others are loose — it lets the closest neighbors speak louder than faraway ones.

06
Dialing Up k

The Bias–Variance Dial

The choice of k is a dial you can turn. Small k (like 1 or 3) means the model listens to only a few neighbors, so it reacts to every odd point — the boundary gets jagged. We call this overfitting: the model is chasing noise. Huge k (like 25+) averages over so many neighbors that it smooths over real patterns — we call this underfitting.

To pick a good k, try several values and see which one classifies the test set most accurately. A handy rule of thumb is to start near the square root of how many training rows you have — with 150 flowers, that's about 12, which is right next door to the 11 we started with.

08
Post-Game

What We Walked Away With

01
Scale Your Features

KNN uses plain straight-line distance to decide who's "close." If one feature has huge numbers and another has tiny ones, the big-numbered feature takes over. Scaling gives every feature a fair vote.

02
Tune k With CV

Small k overreacts to single points; large k smooths everything out. Try a range of values (1 to 30 is usually plenty) and pick the one that gets the most test predictions right.

03
Mind The Weights

Distance weighting makes close neighbors matter more than distant ones. It helps when some groups are packed tighter than others — but a single weird point nearby can throw it off.

KNN isn't the right tool for every job — it gets slow when you have millions of rows, and it struggles when you have lots of features (dozens or more) — but it's a fantastic first algorithm to learn, easy to explain to anyone, and surprisingly competitive on nicely-scaled tables with a few hundred to a few thousand rows.
